# 🦥 Coding Test

## **코딩테스트의 목적**
- 문제 해결 능력 : 문제 이해, **접근 방법(자료구조&알고리즘)**, **코드 설계(시간복잡도)** 
- 구현 능력 : 프로그래밍 언어, 자료구조/알고리즘 구현  
 
**💻코딩테스트 = 자료구조 + 알고리즘💻**

## **문제 풀이 절차**
1. **문제 이해** - 문제 분석하기, 많은 문제 풀어보기    
2. **접근 방법** - 자료구조&알고리즘 이론 학습  
3. **코드 설계** - 시간 복잡도  
4. **코드 구현** - 언어 숙련도, 구현 연습  

## **효율적인 코딩테스트 공부 방법**
1. **시간복잡도**부터 공부하자  
2. 자주 나오는 필수 **자료구조/알고리즘** 위주로 공부  
3. 면접과 코딩테스트에서 필요한 내용은 **다름**  
4. **이론을 제대로 이해**하고 문제 풀이 접근  
5. **구현능력**이 중요, 주력 언어를 정하고 체화 하기  
6. **스터디**를 꼭 하자(문제풀이 공유)    

## **시간 복잡도**  
- 예시  
    - 공이 3, 5, 1, 2, 4가 쓰여진 숫자가 있는데 오름차순으로 정렬하려고 함    
    - 제일 작은 숫자를 선택해 앞으로 옮기기 
    - 남은 공들에 대해서 반복  

1. 시간 크기 비교    
    - **$$O(n!) > O(2^n) > O(n^2) > O(nlogn) > O(n) > O(logn) > O(1)$$**  

    ![image](https://github.com/huniii32/huniii32.github.io/assets/164001121/4bc2785b-9105-45f5-af6e-a9c66746ae3b)  
  
2. Worst Case vs Best Case  
    - 보수적으로 생각 해야함(Worst Case 생각 해보자)  
    - Average Case를 보통 지칭하는데, 구하기가 어려움 → **Worst Case**로 생각함  
3. 코딩 테스트에서의 제약조건
    - 시간복잡도에서 데이터의 크기(n)을 넣어서 나온 값이 100,000,000이 넘으면 시간 제한 초과할 가능성이 있다.  
    - 제약 조건 : $$1 <= n <= 10^5$$  
    - 제약 조건 : $$1 <= n <= 10^8$$
