# 🦥 Coding Test

## **코딩테스트의 목적**
- 문제 해결 능력 : 문제 이해, **접근 방법(자료구조&알고리즘)**, **코드 설계(시간복잡도)** 
- 구현 능력 : 프로그래밍 언어, 자료구조/알고리즘 구현  

**💻코딩테스트 = 자료구조 + 알고리즘💻**

## **문제 풀이 절차**
1. **문제 이해** - 문제 분석하기, 많은 문제 풀어보기   
2. **접근 방법** - 자료구조&알고리즘 이론 학습  
3. **코드 설계** - 시간 복잡도  
4. **코드 구현** - 언어 숙련도, 구현 연습  

## **효율적인 코딩테스트 공부 방법**
1. **시간복잡도**부터 공부하자  
2. 자주 나오는 필수 **자료구조/알고리즘** 위주로 공부  
3. 면접과 코딩테스트에서 필요한 내용은 **다름**  
4. **이론을 제대로 이해**하고 문제 풀이 접근  
5. **구현능력**이 중요, 주력 언어를 정하고 체화 하기  
6. **스터디**를 꼭 하자(문제풀이 공유)   

## **시간 복잡도**  
- 예시  
    - 공이 3, 5, 1, 2, 4가 쓰여진 숫자가 있는데 오름차순으로 정렬하려고 함    
    - 제일 작은 숫자를 선택해 앞으로 옮기기 
    - 남은 공들에 대해서 반복  

1. 시간 크기  
    - **$$O(n!) > O(2^n) > O(n^2) > O(nlogn) > O(n) > O(logn) > O(1)$$**  

    ![image](https://github.com/huniii32/huniii32.github.io/assets/164001121/4bc2785b-9105-45f5-af6e-a9c66746ae3b)  
  
2. Worst Case vs Best Case  
    - 보수적으로 생각 해야함(Worst Case 생각 해보자)  
    - Average Case를 보통 지칭하는데, 구하기가 어려움 → **Worst Case**로 생각함  
3. 코딩 테스트에서의 제약조건
    - 시간복잡도에서 데이터의 크기(n)을 넣어서 나온 값이 100,000,000이 넘으면 시간 제한 초과할 가능성이 있다.  
    $$제약 조건 : 1 <= n <= 10^5$$  
    $$제약 조건 : 1 <= n <= 10^8$$  
    
## **알고리즘**
- 문제해결 방법, 어떠한 문제를 해결하기 위해 정해진 일련의 절차나 방법  
- 자주 쓰이는 문제 해결 방법(알고리즘)은 패턴화  
    - BFS, DFS, Binary Search, Dijkstra 등   
- 한 문제를 해결할 수 있는 알고리즘은 다양함   
    - 각 문제에 적합한 알고리즘을 선택할 수 있어야 함  
    - 알고리즘을 평가할 수 있어야 함  
- 알고리즘 종류(코딩테스트)  
    1. **완전 탐색**
        1. 정답이 될 가능성이 있는 **모든 후보**를 **체계적\(반복문, 재귀, 비트마스크\)**으로 확인하는 것(쉽게 말해 무식한 방법)  
        2. 시간 복잡도가 매우 높아질 수 있음  
        3. 문제의 종류
            - **모든 가능성 탐색** : 모든 가능성을 탐색해보고 문제에서 원하는 결과와 일치하는 값을 찾는 문제  
            - **부분집합 생성** : 모든 부분집합을 생성하여 특정 조건을 만족하는 부분집합을 찾는 문제  
            - **순열과 조합** : 주어진 요소들로 가능한 모든 순열 또는 조합을 생성하는 문제  
            - **격자 탐색** : 격자 내의 모든 위치를 탐색하며 특정 조건을 만족하는 위치를 찾는 문제  
        - (예시) 리스트[4, 9, 7, 5, 1]에서 **두 개**의 숫자를 더해서 **14**가 될 수 있나요? (중복 X)   
  
    2. **재귀**  
        1. 자신을 정의할 때 자기 자신을 재참조하는 함수를 뜻 함  
        2. **시간 복잡도**  
            - 재귀 함수 호출 수 X (재귀 함수 하나당) 시간 복잡도  
        3. **점화식**  
            - 팩토리얼과 피보나치를 자세히 살펴보면 N번째 함수를 N-1번째 함수로 표현 함  
              1. 팩토리얼의 경우 $$f(n) = n * f(n-1)$$  
              2. 피보나치의 경우 $$f(n) = f(n-1) + f(n-2)$$    
        4. **재귀는 자기 자신을 참조하게 되면, 무한 루프에 빠질 수 있음**  → **Base Case**를 설정 해줘야 함  
        5. 문제의 종류 : 팩토리얼, 피보나치  
            - 팩토리얼  
            ```python
            def factorial(n):
              if n == 1:
                return 1
              return n * factorial(n - 1)
            ``` 
            > $$점화식 : f(n) = n * f(n-1)$$    
            > $$Base Case : f(1) = 1$$  
          
            - 피보나치  
            ```python
            def fibo(n):
              if n == 1 or n == 2:
                return 1
              return fibo(n - 1) + fibo(n - 2)
            ``` 
            > $$점화식 : f(n) = f(n-1) + f(n-2)$$  
            > $$Base Case : f(1) = 1, f(2) = 1$$

        **💻재귀 = 점화식 + Base Case💻**  

## 자료구조
1. 데이터를 저장하고 관리하는 방식을 자료구조라고 함  
2. 자료구조 종류(코딩테스트)  
    1. Stack  
    2. Queue    
    3. Graph   
        - **객체**간의 **연결 관계**를 표현하는 자료구조  
        - 객체는 **정점(Node)**, 연결 관계는 **간선(Edge)**로 표현
